基础指针：地址、指向与解引用

从“指针变量保存地址”开始，分清指针、指针变量、取地址和解引用，并建立安全使用指针的基本规则。

← 返回笔记目录 上一篇：指针大小

一、核心认识：指针变量保存地址

可以把指针变量理解为“专门保存地址的变量”，但严格来说，指针是值，指针变量是保存这个值的变量。
#include <stdio.h>

int main(void)
{
    int i = 3;
    int *p;       // p 是变量名，类型是 int *

    p = &i;       // 把 i 的地址保存到 p 中

    printf("%d\n", *p);  // 输出 3
    return 0;
}
指针变量 p 保存内容：&i 保存 i 的起始地址
→
普通变量 i 保存内容：3 地址可以写作 &i
问题：int *p 应该怎样读？

p 是变量名，int * 是它的类型，表示它能够保存 int 对象的地址。

这不代表 p 本身是一个 int，也不代表它当前已经指向了有效对象。

二、p、&i、*p 和 i 的关系

表达式含义在示例中的结果
i普通变量本身3
&i取得变量 i 的地址一个 int * 地址值
p读取指针变量中保存的地址与 &i 相等
*p访问 p 所指向的对象访问的就是 i
 
p = &i;

*p = 10;    // 通过 p 修改 i
printf("%d\n", i);  // 输出 10

i = 20;     // 直接修改 i
printf("%d\n", *p); // 输出 20
更准确的说法是：当 p 有效地指向 i 时，表达式 *p 和表达式 i 都表示同一个对象。它们不是两个对象，也不是说变量 p 等于变量 i。
问题：修改 p、i 和 *p 分别会怎样？

修改 p 是让指针保存另一个地址，一般不会直接修改 i 的值。

修改 i 不会改变 p 保存的地址，但由于 p 仍指向 i，通过 *p 可以看到新值。

修改 *p 就是在修改 p 所指向的对象；此处修改的就是 i。

三、星号 * 的三种常见含义

场景示例含义
乘法运算a * b计算两个数的乘积
声明指针变量int *p;声明一个能够保存 int 地址的变量 p
解引用*p访问 p 所指向的对象
问题：为什么同一个符号会有不同含义？

编译器会根据语法位置判断。int *p; 位于声明中；*p = 5; 位于表达式中；a * b 的星号位于两个操作数之间。

四、指针不是“非负整数”

把指针简单说成“操作受限的非负整数”容易造成误解。指针是独立的数据类型，表示对象或函数的位置；不能把它当作普通整数随意计算。
操作是否允许说明
p + 整数有条件允许只能在同一数组对象及末尾后一个位置的范围内移动
p - 整数有条件允许含义是向前移动若干个同类型元素
p - q有条件允许p、q 必须指向同一数组，或数组末尾后一个位置
p + q不允许两个地址相加没有标准定义
p * q、p / q不允许指针不能做乘法或除法
 
int a[10];
int *p = &a[2];
int *q = &a[7];

q - p;  // 合法，结果为 5，表示相隔 5 个 int 元素
“两个指针指向同一块存储空间才能相减”可以进一步说准确：它们必须指向同一个数组对象中的元素，或者指向该数组末尾后一个位置。

五、未初始化指针与“垃圾值”

int i = 5;
int *p;
int *q;       // 未初始化，值不确定

p = &i;
p = q;        // 错误：把 q 的不确定值复制给 p
printf("%d\n", *q); // 错误：解引用无效指针，行为未定义
局部指针变量没有初始化时，不知道它指向哪里。读取或解引用这种指针都不安全，程序可能崩溃，也可能暂时看似正常；“一定运行到某一行才报错”并不成立。

安全的初始写法

int *q = NULL;  // 明确表示暂时不指向任何对象

if (q != NULL) {
    printf("%d\n", *q);
}
问题：为什么已经声明成功，却仍然不能使用 *q？

声明指针变量只为指针本身分配了空间，并没有自动为它准备一个可访问的目标对象。只有当 q 保存了某个有效对象的地址之后，才能安全解引用。

六、实参、形参与通过指针修改变量

实参是调用函数时提供的表达式；形参是被调用函数中接收实参值的新变量。C 语言的参数传递本质上都是值传递。
void set_value(int *value)
{
    *value = 100;  // 修改 value 指向的对象
}

int main(void)
{
    int number = 3;

    set_value(&number); // 实参是 number 的地址
    printf("%d\n", number); // 输出 100
    return 0;
}
位置内容作用
调用处&number实参，计算出 number 的地址并传入函数
函数定义int *value形参，接收传入地址的一份副本
函数内部*value = 100沿着地址找到并修改 number
问题：既然是值传递，为什么函数还能修改主函数变量？

函数得到的是地址值的一份副本，但副本和原地址指向同一个对象。通过解引用这个地址副本，函数仍然能够访问并修改原对象。

七、指针为什么重要？

构造复杂数据结构链表、树、图等结构依靠指针连接不同节点。
减少大对象复制传递对象地址可以避免复制整块数据，但要明确生命周期和修改权限。
让函数产生多个输出函数可以通过多个输出指针修改调用者提供的对象。
访问硬件和映射内存嵌入式系统中常用指针访问寄存器或共享内存。
处理数组和字符串C 字符串与数组操作大量使用指针遍历。
理解更高层抽象C++ 的引用、迭代器和智能指针都建立在相近的对象访问思想上。

九、最后总结

指针变量保存地址，& 用于取得地址，* 用于访问所指对象。使用指针前必须确保它保存的是有效地址，并且对象仍然处于生命周期内。
核心记忆：地址要有效，类型要匹配，边界和生命周期要清楚。
