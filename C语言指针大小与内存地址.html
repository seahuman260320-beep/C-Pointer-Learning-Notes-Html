指针大小与内存地址

理解指针保存什么、为什么不同类型的指针通常一样大，以及指针类型真正影响了什么。

一、先记住结论

指针中保存的是对象的起始地址。指针大小主要由目标平台决定，不由它指向的数据类型决定。

二、不同类型的指针保存什么？

char   c = 'A';
int    i = 100;
double d = 3.14;

char   *p = &c;
int    *q = &i;
double *r = &d;
变量 通常占用 对应指针 指针保存的内容
c1 字节pc 的起始地址
i4 字节qi 的起始地址
d8 字节rd 的起始地址
问题：为什么 p、q、r 通常一样大？

因为它们保存的都是内存地址。虽然地址指向的数据大小不同，但同一平台通常使用相同宽度来表示对象地址。

例如在 64 位程序中，它们通常都占 8 字节；在 32 位程序中，它们通常都占 4 字节。

三、指针一定占 4 字节吗？

“指针变量本身只占 4 字节”只适用于常见的 32 位环境，并不是通用规则。
答案：指针大小取决于编译目标
程序环境指针通常大小原因
32 位4 字节地址通常由 32 个二进制位表示
64 位8 字节地址通常由 64 个二进制位表示
 
printf("%zu\n", sizeof(p));
printf("%zu\n", sizeof(q));
printf("%zu\n", sizeof(r));

问题：指针只保存首地址，编译器怎么知道要读取几个字节？

编译器通过指针类型判断。例如 int * 解引用时按照 int 的大小和规则读取，double * 则按照 double 读取。

五、指针类型真正影响什么？

指针类型通常不改变指针本身的大小，但会影响解引用方式和指针运算的步长。

p + 1;  // char*   通常向后移动 1 字节
q + 1;  // int*    通常向后移动 4 字节
r + 1;  // double* 通常向后移动 8 字节
问题：为什么“指针加一”移动的字节数不同？

因为“加一”的含义是移动到下一个同类型元素，而不是单纯把地址数值增加 1。

如果 q 指向一个 int，那么 q + 1 应该越过当前整个 int，到达下一个 int。

六、容易混淆的两个 sizeof

sizeof(q)   // q 这个指针变量本身的大小
sizeof(*q)  // q 所指向的 int 数据的大小
示例：在常见 64 位环境中结果可能是多少？

sizeof(q) 通常为 8，因为指针占 8 字节。

sizeof(*q) 通常为 4，因为 *q 的类型是 int。

核心记忆：指针存地址，类型管解释，平台定大小。
