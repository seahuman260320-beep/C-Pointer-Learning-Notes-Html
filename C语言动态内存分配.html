动态内存分配

一、传统数组的四个局限

普通局部数组通常在进入作用域时确定大小，适合长度明确、生命周期简单的数据。

局限说明
长度需要提前确定固定长度数组不能在运行过程中扩容或缩小。
存储空间自动管理离开数组所在的作用域后，不能再访问该局部数组。
生命周期受作用域限制函数内的局部数组不能直接交给调用者长期保存。
可能浪费或不足预留过大浪费空间，预留过小又无法容纳实际数据。
 
int fixed[3] = {1, 2, 3}; // 长度固定

// int n = 5;
// int a[n]; // C99 支持变长数组，但它仍不能动态扩容，生命周期也受作用域限制
变长数组并不等于动态分配。它通常仍具有自动存储期，离开作用域后就失效。

二、为什么需要动态分配内存？

当数据量只能在运行时确定，或对象需要在创建它的函数结束后继续存在时，可以向动态存储区申请空间。

确定需求运行时得到元素个数
→
申请并使用malloc 返回首地址
→
主动释放free 归还内存
动态内存解决的是“大小在运行时决定”和“生命周期由程序控制”这两个核心问题。

三、认识 malloc 与 free

malloc 声明在标准头文件 <stdlib.h> 中，参数是要申请的字节数，成功时返回内存首地址，失败时返回空指针。

#include <stdio.h>
#include <stdlib.h>

int main(void)
{
    int *p = malloc(sizeof *p);
    if (p == NULL) {
        return 1;
    }

    *p = 5;
    printf("%d\n", *p);

    free(p);
    p = NULL;
    return 0;
}
表达式含义
int *p指针变量本身，通常具有自动存储期。
malloc(sizeof *p)申请足以存放一个 int 的空间。
*p = 5通过指针访问动态分配的对象。
free(p)释放 p 指向的动态内存，不会销毁变量 p 本身。
为什么不写 (int *)malloc(4)？

标准 C 中 void * 会自动转换为对象指针，不需要强制转换。使用 sizeof *p 也比写死 4 更可移植，因为标准没有规定 int 必须占 4 字节。

释放后不能继续解引用原地址。将 p 设为 NULL 可以减少误用，但其他指向同一地址的指针仍会成为悬空指针。

四、构造一个动态数组

#include <stdint.h>
#include <stdio.h>
#include <stdlib.h>

int main(void)
{
    size_t len;
    printf("请输入数组长度：");
    if (scanf("%zu", &len) != 1 || len == 0) {
        return 1;
    }

    int *arr;
    if (len > SIZE_MAX / sizeof *arr) {
        return 1;
    }

    arr = malloc(len * sizeof *arr);
    if (arr == NULL) {
        return 1;
    }

    for (size_t i = 0; i < len; ++i) {
        arr[i] = (int)(i + 1);
    }

    for (size_t i = 0; i < len; ++i) {
        printf("%d%c", arr[i], i + 1 == len ? '\n' : ' ');
    }

    free(arr);
    arr = NULL;
    return 0;
}

arr[i] 仍然等价于 *(arr + i)。动态数组和普通数组都能使用下标，但动态数组的长度必须由程序另外保存。

长度来自输入时，必须在乘法和内存分配之前检查 len > SIZE_MAX / sizeof *arr，防止字节数计算溢出后分配到过小的空间。

五、自动存储期与动态内存比较

比较项局部变量/数组动态分配内存
创建方式通过变量声明创建通过 malloc 等函数申请
释放方式离开作用域时自动结束生命周期程序调用 free 主动释放
大小声明时确定；变长数组也不能扩容可根据运行时数据决定
常见实现区域通常位于栈中通常位于堆中
主要风险返回局部对象地址会产生无效指针内存泄漏、重复释放和释放后使用
“栈”和“堆”是常见实现概念；C 标准更准确的表述是自动存储期与动态分配对象。

六、多级指针表示逐级保存地址

int value = 10;
int *p = &value;
int **q = &p;
int ***r = &q;

printf("%d\n", ***r); // 输出 10
变量保存的内容解引用结果
pvalue 的地址*p 是 value
qp 的地址*q 是 p
rq 的地址***r 是 value
r = &p 类型不匹配：r 需要保存 int ** 变量的地址，而 &p 的类型是 int **，不是 int *** 所需的指向对象类型。

七、跨函数使用内存：修改调用者的指针

函数参数按值传递。若函数需要让调用者的指针指向新分配的内存，可以传入该指针的地址，即使用二级指针。

#include <stdio.h>
#include <stdlib.h>

int create_number(int **out)
{
    *out = malloc(sizeof **out);
    if (*out == NULL) {
        return 0;
    }

    **out = 5;
    return 1;
}

int main(void)
{
    int *p = NULL;

    if (!create_number(&p)) {
        return 1;
    }

    printf("%d\n", *p);
    free(p);
    p = NULL;
    return 0;
}
为什么不能返回局部变量的地址？ 
int *wrong(void)
{
    int value = 5;
    return &value; // 错误：函数结束后 value 的生命周期结束
}

局部变量失效后，返回的地址不再指向有效对象。动态分配的对象不会因为创建它的函数返回而自动消失，因此可以交给调用者，但调用者最终必须负责释放。

还可以直接返回动态内存地址吗？

可以。函数可以返回 int *，失败时返回 NULL。二级指针适合函数还需要用返回值表达成功状态，或同时输出多个结果的情况。

八、最后总结

动态内存的完整责任链是：计算大小、申请空间、检查结果、在合法边界内使用、明确所有权，最后恰好释放一次。
要点推荐写法
头文件#include <stdlib.h>
申请大小malloc(count * sizeof *p)
失败检查使用前判断 p == NULL
释放free(p); p = NULL;
跨函数修改指针传入指针地址，形参使用二级指针
核心记忆：按需申请，检查结果，明确所有权，用完释放。
