C语言 · 指针基础笔记 03

数组名、指针与下标运算

理解数组名为什么经常表现得像指针，以及 p[i] 为什么能够访问数组元素。

← 返回笔记目录 ← 上一篇：指针大小 下一篇：动态内存 →

一、一维数组保存一组同类型元素

int a[5] = {1, 2, 3, 4, 5};

这条语句创建了一个包含 5 个 int 元素的一维数组。数组元素在内存中按顺序连续排列。

1a[0]
2a[1]
3a[2]
4a[3]
5a[4]
理解一个数组至少要关注元素类型、数组名和元素个数。这里的类型是 int，名字是 a，长度是 5。

二、数组名不是指针变量

“数组名是指针常量”是便于入门的说法，但不够准确。数组名表示数组对象；在大多数表达式中，数组表达式会自动转换为指向首元素的指针。 
int a[5] = {1, 2, 3, 4, 5};
int *p = a;  // 此处 a 转换为 &a[0]
表达式常见含义类型
a在多数表达式中转换为首元素地址int *
&a[0]第一个元素的地址int *
&a整个数组的地址int (*)[5]
p保存首元素地址的指针变量int *
问题：既然 a 经常等于 &a[0]，为什么不能写 a = p;？

因为 a 是数组名，不是一个可以重新赋值的指针变量。数组创建后，它所代表的那块数组存储区域不能通过给数组名赋值来改变。

p 才是指针变量，可以保存另一个合适的地址。

三、数组和指针的常用等价写法

int a[5] = {1, 2, 3, 4, 5};
int *p = a;

a        == &a[0]
*a       == a[0]
a + 1    == &a[1]
*(a + 1) == a[1]
a[i]     == *(a + i)
p[i]     == *(p + i)

这些等价关系成立的核心是：加法按照所指元素类型移动。对于 int *，加 1 表示移动到下一个 int 元素。

问题：a + 1 是地址增加 1 个字节吗？

不是。它移动一个 int 元素的距离，通常是 sizeof(int) 个字节。指针运算的单位是元素，不是字节。

四、为什么指针可以写成 p[2]？

在 C 语言中，下标表达式按照下面的规则解释：

p[i] 等价于 *(p + i)
int a[3] = {10, 20, 30};
int *p = a;

p[2] = 300;

上面的赋值可以逐步展开：

p + 2          // 得到 a[2] 的地址
*(p + 2)       // 访问 a[2]
*(p + 2) = 300 // 修改 a[2]

// 因此：
p[2] = 300;
表达式访问对象初始值
p、p + 0a[0]10
p + 1a[1]20
p + 2a[2]30

五、完整示例程序

#include <stdio.h>

int main(void)
{
    int a[3] = {10, 20, 30};
    int *p = a;

    *p = 100;       // 修改 a[0]
    *(p + 1) = 200; // 修改 a[1]
    p[2] = 300;     // 等价于 *(p + 2) = 300

    printf("%d %d %d\n", a[0], a[1], a[2]);
    return 0;
}

程序输出：

100 200 300
问题：这里的 p 是数组吗？

不是。p 是指针变量，只保存了 a[0] 的地址。但由于数组元素连续存储，p 可以在合法边界内通过指针运算访问后续元素。

六、使用下标必须遵守数组边界

int a[3] = {10, 20, 30};
int *p = a;

p[0]; // 正确，对应 a[0]
p[1]; // 正确，对应 a[1]
p[2]; // 正确，对应 a[2]
p[3]; // 错误，超过数组范围
越界访问属于未定义行为。程序可能崩溃、读到垃圾值、破坏其他数据，也可能暂时看起来正常；“没有立即报错”不代表代码正确。

七、数组和指针相似，但不是同一种对象

比较项数组 a指针变量 p
本质包含多个元素的数组对象保存地址的变量
能否重新赋值不能写 a = p可以写 p = other
sizeofsizeof(a) 是整个数组大小sizeof(p) 是指针本身大小
下标访问a[i]p[i]，前提是指向有效连续元素
 
int a[5];
int *p = a;

sizeof(a); // 通常是 5 * sizeof(int)
sizeof(p); // 指针大小，常见为 4 或 8 字节
方括号并不是数组对象独有的语法。只要指针指向合法的连续元素区域，就可以使用下标；但它不会自动检查边界。

八、最后总结

数组名不是指针变量，但在大多数表达式中会转换为首元素指针；下标运算 x[i] 本质上等价于 *(x + i)，所有访问都必须位于有效数组边界内。
核心记忆：数组名会转换，指针可以下标，访问不能越界。
